work done till now:
 We switched from Mistral 7B to Phi-2 because Mistral was too heavy and unstable on a 12GB RTX 3060 (Windows),
 while Phi-2 is smaller, stable, and still deep enough (32 layers) for proper sensitivity analysis. 
 We evaluated the model using perplexity on WikiText-2 since it measures how well the model predicts the next word (lower is better). 
 Sensitivity is measured by checking how much perplexity increases when a layer is modified compared to baseline. 
 Masking an entire layer caused perplexity to explode because each transformer block is essential for maintaining information flow through residual connections. 
 We couldn’t inject noise directly since the model was loaded in 4-bit, meaning weights are stored in quantized format (not normal float tensors). 
 Each transformer layer mainly consists of self-attention and an MLP block. 
 Gaussian noise is a better simulation of quantization than zeroing because quantization introduces small perturbations, not complete removal. 
 Highly sensitive layers should be kept in higher precision in mixed precision setups. Next step is to test attention vs MLP sensitivity or try precision swapping per layer. 
 High perplexity indicates poor prediction quality and higher uncertainty.

Attention PPL: 8.77

Early MLP is extremely important
Removing it collapses representations immediately
Attention in Layer 0 is less catastrophic than MLP


Middle Layers Are Surprisingly Stable
Layers 1–26 mostly:
Attention PPL ≈ 6.7–7.3
MLP PPL ≈ 6.8–7.4
Masking either component only slightly increases perplexity
Redundancy exists
Model distributes computation across layers

Later Layers Slightly More Sensitive (Attention)
Layer 27 Attention: 7.34
Layer 31 MLP: 7.89
Early feature expansion (Layer 0 MLP) is foundational.
Middle layers are distributed and robust.
Late layers refine output probabilities.
Catastrophic behavior happens only when full layer removed, not component-level.

insights:
8-bit quantization is almost lossless
NF4 preserves model distribution significantly better than FP4
Quantization does not guarantee speedup on small GPUs
Peak memory depends on kernel implementation, not just bit width
FP16 can outperform quantized models in throughput on consumer GPUs
